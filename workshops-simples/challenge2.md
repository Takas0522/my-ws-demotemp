# 2.アプリケーションを調査する

## 目的

アプリケーションの構造がわかりません。モダナイゼーションを進めていくうえでアプリケーションの構造を把握することは非常に重要です。現在のアプリケーションの状態を把握しドキュメントとして残し、人とAIが活用できる資材として残します。マイクロサービスもどきの構造や内部で使用されているフレームワーク、データ構造などを明らかにしていきます。

## 手段

GitHub Copilotを使用してアプリケーションの構造を明らかにしていきます。特定された情報をドキュメント化すれば後任の方々やAIのためになります。

ヒントになりそうなキーワード
* Mermaid記法:シーケンス図の作成などを依頼したい場合に
* 初期データ:DB初期化のためのデータがあるようです
* ユニットテスト/インテグレーションテスト/e2eテスト:テストのひな型はすでに存在するようです
* サービス間連携:どのような手法をとっているかわかりませんが、このアプリケーションは他サービスと連携して一つのアプリとして提供されているようです
* /docs: リポジトリ内のアプリをここに格納しようとしたのが見て取れます
